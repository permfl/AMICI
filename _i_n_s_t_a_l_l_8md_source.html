3 ## Availability
4 
5 The sources for AMICI are accessible as
6 - Source [tarball](https://github.com/ICB-DCM/AMICI/tarball/master)
7 - Source [zip](https://github.com/ICB-DCM/AMICI/zipball/master)
8 - GIT repository on [github](https://github.com/ICB-DCM/AMICI)
9 
10 ### Obtaining AMICI via the GIT versioning system
11 In order to always stay up-to-date with the latest AMICI versions, simply pull it from our GIT repository and
12 recompile it when a new release is available. For more information about GIT checkout their [website](http://git-scm.com/)
13 
14 The GIT repository can currently be found at https://github.com/ICB-DCM/AMICI and a direct clone is possible via
15 
16  git clone https://github.com/ICB-DCM/AMICI.git AMICI
17 
18 ## Installation
19 
20 If AMICI was downloaded as a zip, it needs to be unpacked in a convenient directory. If AMICI was obtained via cloning of the git repository, no further unpacking is necessary.
21 
22 ### Dependencies
23 
24 The MATLAB interface only depends on the symbolic toolbox, which is needed for model compilation, but not simulation.
25 
26 #### Symbolic Engine
27 
28 The MATLAB interface requires the symbolic toolbox for model compilation. The symbolic toolbox is not required for model simulation.
29 
30 #### Math Kernel Library (MKL)
31 
32 The python and C++ interfaces require a system installation of `BLAS`. AMICI has been tested with various native and general purpose MKL implementations such as
33 Accelerate, Intel MKL, cblas, openblas, atlas. The matlab interface uses the MATLAB MKL, which requires no prior installation.
34 
35 #### HDF5
36 
37 The python and C++ interfaces provide routines to read and write options and results in [hdf5](https://support.hdfgroup.org/HDF5/) format. For the python interface, the installation of hdf5 is optional, but for the C++ interace it is required.
38 HDF can be installed using package managers such as [brew](https://brew.sh) or [apt](https://wiki.debian.org/Apt):
39 
40  brew install hdf5
41 
42 or
43 
44  apt-get install libhdf5-serial-dev
45 
46 
47 #### SWIG
48 
49 The python interface requires [SWIG](http://www.swig.org), which has to be installed by the user.
50 Swig can be installed using package managers such as [brew](https://brew.sh) or [apt](https://wiki.debian.org/Apt):
51 
52  brew install swig
53 
54 or
55 
56  apt-get install swig3.0
57 
58 We note here that some linux package managers may provide swig executables as `swig3.0`, but installation as `swig` is required. This can be fixed using, e.g., symbolic links:
59 
60  mkdir -p ~/bin/ && ln -s $(which swig3.0) ~/bin/swig && export PATH=~/bin/:$PATH
61 
62 #### python packages
63 
64 The python interface requires the python packages `pkgconfig` and `numpy` to be installed before AMICI can be installed. These can be installed via `pip`:
65 
66  pip3 install pkgconfig numpy
67 
68 ### MATLAB
69 
70 To use AMICI from MATLAB, start MATLAB and add the AMICI/matlab direcory to the MATLAB path. To add all toolbox directories to the MATLAB path, execute the matlab script
71 
72  installAMICI.m
73 
74 To store the installation for further MATLAB session, the path can be saved via
75 
76  savepath
77 
78 For the compilation of .mex files, MATLAB needs to be configured with a working C compiler. The C compiler needs to be installed and configured via:
79 
80  mex -setup c
81 
82 For a list of supported compilers we refer to the mathworks documentation: [mathworks.com](http://mathworks.com/support/compilers/R2018b/index.html)
83 Note that Microsoft Visual Studio compilers are currently not supported.
84 
85 ### python
86 
87 To use AMICI from python, install the module and all other requirements using pip
88 
89  pip3 install amici
90 
91 You can now import it as python module:
92 
93  import amici
94 
95 ### C++
96 
97 To use AMICI from C++, run the
98 
99  ./scripts/buildSundials.sh
100  ./scripts/buildSuitesparse.sh
101  ./scripts/buildAmici.sh
102 
103 script to compile amici libary. The static library file can then be linked from
104 
105  ./build/libamici.a
106 
107 In CMake-based packages, amici can be linked via
108 
109  find_package(Amici)

111 ## Dependencies
112 
113 The MATLAB interface requires the Mathworks Symbolic Toolbox for model generation via `amiwrap(...)`, but not for execution of precompiled models. Currently MATLAB R2018a or newer is not supported (see https://github.com/ICB-DCM/AMICI/issues/307)
114 
115 The python interface requires python 3.6 or newer and `cblas` library to be installed. Windows installations via pip are currently not supported, but users may try to install amici using the build scripts provided for the C++ interface (these will by default automatically install the python module).
116 
117 The C++ interface requires `cmake` and `cblas` to be installed.
118 
119 The tools SUNDIALS and SuiteSparse shipped with AMICI do __not__ require explicit installation.
120 
121 AMICI uses the following packages from SUNDIALS:
122 
123 __CVODES__: the sensitivity-enabled ODE solver in SUNDIALS. Radu Serban and Alan C. Hindmarsh. _ASME 2005 International Design Engineering Technical Conferences and Computers and Information in Engineering Conference._ American Society of Mechanical Engineers, 2005. [PDF](http://proceedings.asmedigitalcollection.asme.org/proceeding.aspx?articleid=1588657)
124 
125 __IDAS__
126 
127 AMICI uses the following packages from SuiteSparse:
128 
129 __Algorithm 907: KLU__, A Direct Sparse Solver for Circuit Simulation Problems. Timothy A. Davis, Ekanathan Palamadai Natarajan, _ACM Transactions on Mathematical Software_, Vol 37, Issue 6, 2010, pp 36:1 - 36:17. [PDF](http://dl.acm.org/authorize?305534)
130 
131 __Algorithm 837: AMD__, an approximate minimum degree ordering algorithm, Patrick R. Amestoy, Timothy A. Davis, Iain S. Duff, _ACM Transactions on Mathematical Software_, Vol 30, Issue 3, 2004, pp 381 - 388. [PDF](http://dl.acm.org/authorize?733169)
132 
133 __Algorithm 836: COLAMD__, a column approximate minimum degree ordering algorithm, Timothy A. Davis, John R. Gilbert, Stefan I. Larimore, Esmond G. Ng
134 _ACM Transactions on Mathematical Software_, Vol 30, Issue 3, 2004, pp 377 - 380. [PDF](http://dl.acm.org/authorize?734450)
