1 # MATLAB Interface {#matlab_interface}
2 
3 In the following we will give a detailed overview how to specify models in MATLAB and how to call the generated simulation files.
4 
5 ## Model Definition
6 
7 This guide will guide the user on how to specify models in MATLAB. For example implementations see the examples in the matlab/examples directory.
8 
9 ### Header
10 
11 The model definition needs to be defined as a function which returns a struct with all symbolic definitions and options.
12 
13  function [model] = example_model_syms()
14 
15 ### Options
16 
17 Set the options by specifying the respective field of the modelstruct
18 
19  model.(fieldname) = value
20 
21 The options specify default options for simulation, parametrisation and compilation. All of these options are optional.
22 
23 | field | description | default
24 |--------------|-----------------------------------------------|---------
25 | .param | default parametrisation 'log'/'log10'/'lin' | 'lin'
26 | .debug | flag to compile with debug symbols | false
27 | .forward | flag to activate forward sensitivities | true
28 | .adjoint | flag to activate adjoint sensitivities | true
29 
30 When set to false, the fields 'forward' and 'adjoint' will speed up the time required to compile the model but also disable the respective sensitivity computation.
31 
32 ### States
33 
34 Create the respective symbolic variables. The name of the symbolic variable can be chosen arbitrarily.
35 
36  syms state1 state2 state3
37 
38 Create the state vector containing all states:
39 
40  model.sym.x = [ state1 state2 state3 ];
41 
42 ### Parameters
43 
44 Create the respective symbolic variables. The name of the symbolic variable can be chosen arbitrarily.
45 Sensitivities __will be derived__ for all paramaters.
46 
47  syms param1 param2 param3 param4 param5 param6
48 
49 Create the parameters vector
50 
51  model.sym.p = [ param1 param2 param3 param4 param5 param6 ];
52 
53 ### Constants
54 
55 Create the respective symbolic variables. The name of the symbolic variable can be chosen arbitrarily.
56 Sensitivities with respect to constants __will not be derived__.
57 
58  syms const1 const2
59 
60 Create the parameters vector
61 
62  model.sym.k = [ const1 const2 ];
63 
64 ### Differential Equation
65 
66 For time-dependent differential equations you can specify a symbolic variable for time. This __needs__ to be denoted by `t`.
67 
68  syms t
69 
70 Specify the right hand side of the differential equation `f` or `xdot`
71 
72  model.sym.xdot(1) = [ const1 - param1*state1 ];
73  model.sym.xdot(2) = [ +param2*state1 + dirac(t-param3) - const2*state2 ];
74  model.sym.xdot(3) = [ param4*state2 ];
75 
76 or
77 
78  model.sym.f(1) = [ const1 - param1*state1 ];
79  model.sym.f(2) = [ +param2*state1 + dirac(t-param3) - const2*state2 ];
80  model.sym.f(3) = [ param4*state2 ];
81 
82 The specification of `f` or `xdot` may depend on states, parameters and constants.
83 
84 For DAEs also specify the mass matrix.
85 
86  model.sym.M = [1, 0, 0;...
87  0, 1, 0;...
88  0, 0, 0];
89 
90 The specification of M may depend on parameters and constants.
91 
92 For ODEs the integrator will solve the equation \f$ \dot{x} = f \f$ and for DAEs the equations \f$ M \cdot \dot{x} = f \f$.
93 AMICI will decide whether to use CVODES (for ODEs) or IDAS (for DAEs) based on whether the mass matrix is defined or not.
94 
95 In the definition of the differential equation you can use certain symbolic functions. For a full list of available functions see `src/symbolic_functions.cpp`.
96 
97 Dirac functions can be used to cause a jump in the respective states at the specified time-point. This is typically used to model injections, or other external stimuli. Spline functions can be used to model time/state dependent response with unkown time/state dependence.
98 
99 ### Initial Conditions
100 
101 Specify the initial conditions. These may depend on parameters on constants and must have the same size as `x`.
102 
103  model.sym.x0 = [ param4, 0, 0 ];
104 
105 ### Observables
106 
107 Specify the observables. These may depend on parameters and constants.
108 
109  model.sym.y(1) = state1 + state2;
110  model.sym.y(2) = state3 - state2;
111 
112 In the definition of the observable you can use certain symbolic functions. For a full list of available functions see `src/symbolic_functions.cpp`.
113 Dirac functions in observables will have no effect.
114 
115 ### Events
116 
117 Specifying events is optional. Events are specified in terms of a trigger function, a bolus fuction and an output function. The roots of the trigger function defines the occurences of the event. The bolus function defines the change in the state on event occurences. The output function defines the expression which is evaluated and reported by the simulation routine on every event occurence. The user can create events by constructing a vector of objects of the class @ref amievent.
118 
119  model.sym.event(1) = amievent(state1 - state2,0,[]);
120 
121 Events may depend on states, parameters and constants but __not__ on observables.
122 
123 For more details about event support see https://doi.org/10.1093/bioinformatics/btw764
124 
125 ### Standard Deviation
126 
127 Specifying standard deviations is optional. It only has an effect when computing adjoint sensitivities. It allows the user to specify standard deviations of experimental data for observables and events.
128 
129 Standard deviaton for observable data is denoted by sigma_y
130 
131  model.sym.sigma_y(1) = param5;
132 
133 Standard deviaton for event data is denoted by sigma_t
134 
135  model.sym.sigma_t(1) = param6;
136 
137 Both `sigma_y` and `sigma_t` can either be a scalar or of the same dimension as the observables / events function.
138 They can depend on time and parameters but must not depend on the states or observables. The values provided in `sigma_y` and `sigma_t` will only be used if the value in `D.Sigma_Y` or `D.Sigma_T` in the user-provided data struct is `NaN`. See simulation for details.
139 
140 ### Objective Function
141 
142 By default, AMICI assumes a normal noise model and uses the corresponding negative log-likelihood
143 
144  J = 1/2*sum(((y_i(t)-my_ti)/sigma_y_i)^2 + log(2*pi*sigma_y^2)
145 
146 as objective function. A user provided objective function can be specified in
147 
148  model.sym.Jy
149 
150 As reference see the default specification of `this.sym.Jy` in `amimodel.makeSyms`.
151 
152 ### SBML
153 
154 AMICI can also import SBML models using the command `SBML2AMICI`. This will generate a model specification as described above, which may be edited by the user to apply further changes.
155 
156 ## Model Compilation
157 
158 The model can then be compiled by calling `amiwrap.m`:
159 
160  amiwrap(modelname,'example_model_syms',dir,o2flag)
161 
162 Here `modelname` should be a string defining the name of the model, `dir` should be a string containing the path to the directory in which simulation files should be placed and `o2flag` is a flag indicating whether second order sensitivities should also be compiled.
163 The user should make sure that the previously defined function `'example_model_syms'` is in the user path. Alternatively, the user can also call the function `'example_model_syms'`
164 
165  [model] = example_model_syms()
166 
167 and subsequently provide the generated struct to `amiwrap(...)`, instead of providing the symbolic function:
168 
169  amiwrap(modelname,model,dir,o2flag)
170 
171 In a similar fashion, the user could also generate multiple models and pass them directly to `amiwrap(...)` without generating respective model definition scripts.
172 
173 ## Model Simulation
174 
175 After the call to `amiwrap(...)` two files will be placed in the specified directory. One is a _modelname_.mex and the other is simulate_ _modelname_.m. The mex file should never be called directly. Instead the MATLAB script, which acts as a wrapper around the .mex simulation file should be used.
176 
177 The simulate_ _modelname_.m itself carries extensive documentation on how to call the function, what it returns and what additional options can be specified. In the following we will give a short overview of possible function calls.
178 
179 ### Integration
180 
181 Define a time vector:
182 
183  t = linspace(0,10,100)
184 
185 
186 Generate a parameter vector:
187 
188  theta = ones(6,1);
189 
190 
191 Generate a constants vector:
192 
193  kappa = ones(2,1);
194 
195 
196 Integrate:
197 
198  sol = simulate_modelname(t,theta,kappa,[],options)
199 
200 
201 The integration status will be indicated by the `sol.status` flag. Negative values indicated failed integration. The states will then be available as sol.x. The observables will then be available as `sol.y`. The event outputs will then be available as `sol.z`. If no event occured there will be an event at the end of the considered interval with the final value of the root function is stored in `sol.rz`.
202 
203 Alternatively the integration can also be called via
204 
205  [status,t,x,y] = simulate_modelname(t,theta,kappa,[],options)
206 
207 The integration status will be indicated by the flag `status` . Negative values indicated failed integration. The states will then be available as `x`. The observables will then be available as `y`. No event output will be given.
208 
209 ### Forward Sensitivities
210 
211 Set the sensitivity computation to forward sensitivities and integrate:
212 
213  options.sensi = 1;
214  options.sensi_meth = 'forward;
215  sol = simulate_modelname(t,theta,kappa,[],options)
216 
217 The integration status will be indicated by the `sol.status` flag. Negative values indicate failed integration. The states will be available as `sol.x`, with the derivative with respect to the parameters in `sol.sx`. The observables will be available as `sol.y`, with the derivative with respect to the parameters in `sol.sy`. The event outputs will be available as `sol.z`, with the derivative with respect to the parameters in `sol.sz`. If no event occured there will be an event at the end of the considered interval with the final value of the root function stored in `sol.rz`, with the derivative with respect to the parameters in `sol.srz`.
218 
219 Alternatively the integration can also be called via
220 
221  [status,t,x,y,sx,sy] = simulate_modelname(t,theta,kappa,[],options)
222 
223 The integration status will be indicated by the status flag. Negative values indicate failed integration. The states will then be available as `x`, with derivative with respect to the parameters in `sx`. The observables will then be available as `y`, with derivative with respect to the parameters in `sy`. No event output will be given.
224 
225 ### Adjoint Sensitivities
226 
227 Set the sensitivity computation to adjoint sensitivities:
228 
229  options.sensi = 1;
230  options.sensi_meth = 'adjoint';
231 
232 Define Experimental Data:
233 
234  D.Y = [NaN(1,2)],ones(length(t)-1,2)];
235  D.Sigma_Y = [0.1*ones(length(t)-1,2),NaN(1,2)];
236  D.T = ones(1,1);
237  D.Sigma_T = NaN;
238 
239 The `NaN` values in `Sigma_Y` and `Sigma_T` will be replaced by the specification in `model.sym.sigma_y` and `model.sym.sigma_t`. Data points with `NaN` value will be completely ignored.
240 
241 Integrate:
242 
243  sol = simulate_modelname(t,theta,kappa,D,options)
244 
245 The integration status will be indicated by the sol.status flag. Negative values indicate failed integration. The log-likelihood will then be available as `sol.llh` and the derivative with respect to the parameters in `sol.sllh`. Notice that for adjoint sensitivities no state, observable and event sensitivities will be available. Yet this approach can be expected to be significantly faster for systems with a large number of parameters.
246 
247 ### Steady State Sensitivities
248 
249 This will compute state sensitivities according to the formula \f$ s_k^x = -\left(\frac{\partial f}{\partial x} \right)^{-1}\frac{\partial f}{\partial \theta_k} \f$
250 
251 In the current implementation this formulation does not allow for conservation laws as this would result in a singular Jacobian.
252 
253 Set the final timepoint as infinity, this will indicate the solver to compute the steadystate:
254 
255  t = Inf;
256 
257 Set the sensitivity computation to steady state sensitivities:
258 
259  options.sensi = 1;
260 
261 Integrate:
262 
263  sol = simulate_modelname(t,theta,kappa,D,options)
264 
265 The states will be available as `sol.x`, with the derivative with respect to the parameters in `sol.sx`. The observables will be available as `sol.y`, with the derivative with respect to the parameters in `sol.sy`. Notice that for steady state sensitivities no event sensitivities will be available. For the accuracy of the computed derivatives it is essential that the system is sufficiently close to a steady state. This can be checked by examining the right hand side of the system at the final time-point via `sol.diagnosis.xdot`.
